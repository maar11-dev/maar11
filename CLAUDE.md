# Guía de Desarrollo del Portfolio

## 🎯 Objetivo del Proyecto
El usuario está construyendo su propio portfolio web personal basado en una estructura definida

## ⚙️ Directrices Técnicas
* **Estilo Visual**: Queremos un estilo visual muy estético, distintivo e innovador, para ello nos apoyaremos en la skill de frontend.
* **Stack y Skills**: El usuario integrará una "skill" o tecnología que ya tiene preparada. Adapta cualquier solución de código al stack que el usuario te indique usar en cada momento.
* **Despliegue**: El código debe estar preparado para ser desplegado fácilmente (p. ej., GitHub Pages).

## 📂 Estructura del Portfolio
Basado en la referencia proporcionada, el portfolio se organizará en las siguientes secciones. Cada una está pensada para maximizar las oportunidades de conseguir trabajo o prácticas:

### 1. Inicio / Hero
* Presentación impactante.
* Debe dejar claro el rol y la disponibilidad para trabajo/prácticas.

### 2. Sobre Mí
* Contexto personal y profesional.
* Espacio para explicar la situación actual (estudiante/formación) y motivación.

### 3. Proyectos
* El núcleo del portfolio.
* Demostración práctica de las skills. Cada proyecto debe explicar el "qué", "cómo" y "por qué".

### 4. Habilidades / Skills
* Resumen visual de las tecnologías y herramientas dominadas.

### 5. Contacto
* Vías de comunicación directas (Email, LinkedIn, GitHub) con un llamado a la acción claro para reclutadores.
